# Co-Training Vision-Language Models for Remote Sensing Multi-Task Learning

## background
多任务学习方法具有更好的泛化能力、更强的可扩展性和更高的实际应用价值。
地球观测系统已获取了海量的遥感（RS）数据，这迫切要求发展自动化的遥感图像解译技术。可以研发多任务智能体。

## motivation
**大多数遥感图像处理方法通常是在孤立的数据集上训练一个专门设计的模型，以求在单个任务上达到最优性能。**
由于数据和模型架构的异质性，开发一个能够处理多个遥感任务的统一模型，即多任务学习（MTL），仍然具有挑战性。
虽然已经有很多遥感VLM，但每个都有自己的偏重和短板，没有一个真正做到了"全能"（同时擅长语义理解、精准检测和UHR推理）。这正是RSCoVLM要填补的空白。
使用VLM进行遥感多任务学习中，**目前仍然缺乏一个在集成的多任务学习环境中解决这些局限性的统一框架。**
ps：所以需要协同的训练，优势有1.可以同时处理多项任务。2.任务间共享知识，多任务学习缓解了标注数据不足的问题以及过拟合。3.捕捉任务间相关性的联合表征，从而提升泛化能力。

## contribution
提出了RSCoVLM，一个完全开源的用于遥感多任务学习的VLM基线。实验表明，我们的模型在各种数据集和任务的基准测试中均取得了领先性能。

开发了基于VLM的通用遥感多任务学习框架，并创建了数据整理流程，以促进跨各种遥感任务的多数据集统一训练。

提出了适用于遥感的动态分辨率策略，以及“缩放式链式”策略和LRS-VQA-Zoom数据集，以进一步增强模型对超高分辨率图像的推理能力。

开发了一种用于遥感VLM的航空检测方法，并提出了一种评估指标，使得遥感VLM与传统方法之间能够进行公平比较


## method
### 1.通用遥感多任务框架
backbone
遵循主流的VLM范式。它使用视觉编码器和文本分词器分别处理图像和文本输入。随后，基于语言模型的统一解码器处理双模态特征，并执行各种任务，例如遥感图像场景分类、问答、图像描述、视觉定位（grounding）和目标检测。
datapline
包括数据获取、离线处理和在线加载，为模型训练提供多样化的图像及其对应的文本提示和标准答案（golden responses）

### 2.异构数据的处理
| 数据集名称 | 主要用途 / 任务类型 | 具体说明 / 子任务 |
| :--- | :--- | :--- |
| **VHM** | 指令微调（描述相关子集） | 用于图像分类、图像描述、视觉问答等任务，可进一步分解为模态识别、分辨率估计。 |
| **GeoChat** | 指令微调（描述相关子集） | 同上，用于图像分类、图像描述、视觉问答等通用描述性任务。 |
| **refGeo** | 视觉定位 | 作为主要的视觉定位数据源，用于训练模型的指代理解与定位能力。 |
| **TEOChatlas** | 时序多图像理解 | 提供时序多图像数据，用于处理涉及时间序列的遥感图像任务。 |
| **LLaVA-OneVision**（采样部分） | 通用能力保持（持续训练） | 采样其中的图表解读、光学字符识别（OCR）等通用数据，防止模型通用推理能力下降。 |
| **DOTA-v1.0** | 目标检测 | 纳入该数据集以填补遥感视觉语言模型在细粒度目标检测方面的空白，丰富检测相关学习能力。 |
| **自建合成数据（第3.4节详述）** | 缩放式链式能力（Chain-of-Scale） | 基于大规模遥感图像，合成“图像-区域-问题”三元组，用于培养模型的层级化推理能力。 |

所收集数据集的格式和任务要求多样，且存在潜在的系统性噪声，进行了额外的离线预处理，以将所有数据整合到训练框架中。
**首先，** 移除所有任务描述符，即VHM GeoChat中使用到的[grounding][refer][identify]等标签。**这些标签视为一个独立的“单词”，加入到分词器的词表中。**
**接着**,所有边界框，并将其分为水平框、有向框和四边形框。默认下：水平框用于视觉定位任务，而四边形框用于目标检测任务。
**接着**，
对话数据 → OpenAI的messages格式
检测数据 → COCO格式
定位数据 → refGeo格式化，视觉定位数据则遵循refGeo的模式。
**最后**，对于"缩放式链式"数据集，我们应用了工具调用（tool-call）格式。
数据加载使用加权控制。

### 动态解析策略
ps：传统放缩，在右侧或底部补零将图像填充为正方形，然后将其缩放到输入尺寸。仅在固定尺寸或极少数的图像尺寸上对图像进行预处理。
针对此，该策略支持全尺寸输入处理、根据输入分辨率缩放坐标精度，以及整理训练数据以增强跨多分辨率的学习能力。
RSCoVLM的解决思路：动态调整"尺子"的刻度，图像多大，坐标空间就多大。
**首先，全尺寸视觉输入**，比如patch大小为14，一张1020x740尺寸的图片，H=1024/14x14=[72.857]x14=1022;W=⌈740/14⌉×14=⌈52.857⌉×14 =53×14=742.图像会被轻微拉伸/压缩，无需任何填充（padding）或裁剪（crop），保留了图像的原始宽高比和全部信息。

### 超高分辨率遥感图像的放大链
缩放链原理：首先扫视整个图像以确定感兴趣区域（RoI），然后放大这些区域，再执行实际任务。
预测感兴趣区域（RoI）的边界框，然后将该区域裁剪出来，并以原生分辨率（原始高清）输入模型。
>用户 : <提示词> + 图像 + <问题>
助手 : [x1, y1, x2, y2]  (预测感兴趣区域)
用户 : 放大(图像, [x1, y1, x2, y2])  (执行裁剪放大)
助手 : <真实答案>  (最终回答)

模型看到的是下采样后的缩略图（比如把4000×4000缩放到1024×1024）。

模型的任务是：根据问题（比如"图中有多少艘船？"），预测"船最密集的区域"在哪里。

输出是一个水平边界框 [x1, y1, x2, y2]（在缩略图坐标系中）。
系统根据第一步的坐标，在原图上裁剪出对应区域。

将裁剪出的高分辨率区域以原生分辨率输入模型。

模型基于放大后的细节，给出最终答案（如"共有17艘船"）。
数据集：
模板生成（60k）：这是"地基"。通过3×3网格，模型被迫练习"指哪打哪"——不仅要回答问题，还要知道"问题针对的是哪个区域"。这是最纯粹的Zoom-in能力训练。

GPT-4V合成（159k）：这是"血肉"。模板数据太死板（只会问"有几个"），而真实场景中用户会问各种奇怪的问题（"这片区域是城市还是农村？""那艘船是什么颜色的？"）。GPT-4V强大的视觉理解能力帮我们生成了海量高质量、多样化的问答对。

多选查询（83k）：这是"应试技巧"。很多公开基准（如MMBench、SEED-Bench）采用多选形式评估模型。如果不专门训练，模型可能在开放式问答上很强，但在多选上表现不佳。这个子集专门补足了这一短板。
